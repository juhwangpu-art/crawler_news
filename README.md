# 📰 네이버 뉴스 모니터링 대시보드

설정된 키워드에 대해 네이버 뉴스 검색 결과를 **30분 단위로 자동 수집**하고,
기사별 원문 링크 제공 및 **긍정/부정 기획 기사 저장** 기능을 제공하는 Streamlit 대시보드입니다.

## 기능
- 키워드별 네이버 뉴스 자동 수집(앱 실행 중 30분마다 백그라운드 동기화)
- 기사 목록(일시·매체·제목·키워드·요약) + 원문 링크
- 키워드/검색어 필터
- 긍정 기획 기사 저장 — 일시·매체·기자·제목·한줄요약
- 부정 기획 기사 저장 — 동일 항목
- 기자명은 저장 시 기사 본문에서 자동 추출(수정 가능)
- 저장 목록 CSV 다운로드

## 실행
```bat
run.bat
```
또는
```bash
python -m streamlit run app.py
```
브라우저에서 http://localhost:8501 열림.

## 구성 파일
| 파일 | 역할 |
|------|------|
| `config.py` | 키워드 목록, 동기화 주기 설정 |
| `crawler.py` | 네이버 뉴스 검색 스크래핑 + 기자명 추출 |
| `db.py` | SQLite 저장소(`news.db` 자동 생성) |
| `app.py` | Streamlit 대시보드 |

## 키워드 수정
`config.py`의 `KEYWORDS` 리스트를 편집하세요.

## 사용법
1. 앱 실행 → 최초 1회 자동 수집(수집 이력이 없을 때).
2. 사이드바 **`지금 동기화`** 로 수동 수집 가능.
3. `뉴스 피드` 탭에서 기사 행을 선택 → 하단 폼에서 내용 확인/수정 후 **긍정/부정 저장**.
4. `긍정 기획` / `부정 기획` 탭에서 저장 목록 확인·삭제·CSV 다운로드.

## 참고 / 주의
- 네이버 검색 결과 HTML 구조가 바뀌면 `crawler.py`의 CSS 셀렉터를 조정해야 할 수 있습니다.
- 검색 결과에는 기자명이 없어 저장 시 본문에서 추출하며, 언론사별로 실패할 수 있으니 폼에서 직접 보정하세요.
- 자동 동기화는 **앱이 실행 중일 때만** 동작합니다(대시보드 실행 중 자동 방식).
